# Migration 0.2.0 — Adopt pdeqVersion

## Context

Pdeq 0.2.0 introduces the migrations feature itself. Every consumer
project needs a `pdeqVersion` field in `pdeq.json` so future `/migrate`
runs know which version the project's specs and config conform to.
This migration is the on-ramp: it lands the first `pdeqVersion: 0.2.0`
on projects that predate the migrations feature.

Consumers that installed pdeq before 0.2.0 typically have no
`pdeq.json` at all (old `init.sh` only wrote one when non-default
config was needed) or have one without the `pdeqVersion` key. This
migration covers both cases. Projects already running 0.2.0-aware
pdeq are a no-op.

After this migration, the orchestrator's per-migration version bump
writes `pdeqVersion: 0.2.0` authoritatively — the mechanical block
below introduces the field so bump has somewhere to write.

## Mechanical

```shell
# Idempotently ensure pdeq.json exists with a pdeqVersion field.
#
# Three cases:
#   1. File missing              → create with pdeqVersion only.
#   2. File exists, no key       → inject key as first property.
#   3. File exists, key present  → no-op (already migrated or ahead).
#
# PDEQ_CONFIG_PATH lets tests redirect the target; default is ./pdeq.json
# evaluated from the orchestrator's cwd (specsRoot).
set -eu
CONFIG="${PDEQ_CONFIG_PATH:-./pdeq.json}"

if [ ! -f "$CONFIG" ]; then
  printf '{\n  "pdeqVersion": "0.2.0"\n}\n' > "$CONFIG"
  exit 0
fi

# Key already present — any value (including empty string) means the
# field exists; the orchestrator's bump step will overwrite it.
if grep -Eq '^[[:space:]]*"pdeqVersion"[[:space:]]*:' "$CONFIG"; then
  exit 0
fi

# Inject pdeqVersion as the first property after the opening `{`.
# Works whether `{` is on its own line or followed by other content.
awk '
  BEGIN { injected = 0 }
  !injected && /\{/ {
    pos = index($0, "{")
    before = substr($0, 1, pos)
    after  = substr($0, pos + 1)
    if (after ~ /^[[:space:]]*$/) {
      print before
      print "  \"pdeqVersion\": \"0.2.0\","
    } else {
      print before
      print "  \"pdeqVersion\": \"0.2.0\","
      print after
    }
    injected = 1
    next
  }
  { print }
' "$CONFIG" > "$CONFIG.tmp" && mv "$CONFIG.tmp" "$CONFIG"
```

## Notes

The mechanical block deliberately does not validate the resulting
JSON — downstream tooling (schema validation, `scripts/migrate.sh
recorded`) catches malformed config. Authors of future migrations
should prefer `jq` or a similar JSON-aware tool when available; this
migration uses only POSIX utilities because it runs on projects that
may not yet have declared any tooling dependencies.

Idempotency is guaranteed by the three-case branch: any re-run after
a successful apply takes the `grep -Eq` early-exit path. A re-run
after a partial apply (file injected but bump failed) also takes the
early-exit path — bump is the orchestrator's concern, not this
block's.
