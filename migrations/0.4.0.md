# Migration 0.4.0 — Harness-agnostic install layout

## Context

Pdeq 0.4.0 makes pdeq installable into and operable from multiple coding-agent harnesses — Claude Code, Codex CLI, and Pi at v1. The breaking change is in the file layout the framework ships and how `scripts/init.sh` materializes per-harness views in consumer projects. The migration brings an existing 0.3.x consumer's project from the old layout to the new one in a single explicit step. There is no soft transition: after this migration runs (and after the consumer's pinned `.pdeq/` is at 0.4.0), the only supported layout is the 0.4.0 one.

What changes in the consumer's project diff after `/pdeq-migrate` runs:

1. **Per-lane `CLAUDE.md` `@import` targets are rewritten** from `.pdeq/<lane>/CLAUDE.md` to `.pdeq/<lane>/AGENTS.md`. The canonical agent-instructions file inside the submodule is now `AGENTS.md` at every lane; `CLAUDE.md` is no longer present there.

2. **For every non-Claude harness in the consumer's `harnesses` list**, an `AGENTS.md` symlink is created at each lane location pointing at the canonical AGENTS.md inside the submodule. If the consumer hasn't declared `harnesses` in `pdeq.json` yet, the migration treats the list as `["claude"]` (the v0.3.x back-compat default) and writes that field into `pdeq.json` so future tools have a recorded value.

3. **`.claude/commands/pdeq-*.md` symlink targets are re-pointed** from `.pdeq/.claude/commands/pdeq-*.md` to `.pdeq/pdeq-rules/commands/pdeq-*.md`. The slash-command source-of-truth moved inside the submodule to a harness-neutral path.

4. **`.claude/agents/bootstrap-analyzer` and `.claude/agents/bootstrap-generator` symlinks are removed.** Those subagent definition files were Claude-only; their prompts now live inline inside `pdeq-rules/commands/pdeq-bootstrap.md`, so the bootstrap workflow runs from any harness without needing a subagent definition. If either path is a regular file or a non-pdeq symlink (consumer customization), the migration leaves it in place and prints a one-line stderr warning naming the file.

Files outside the four touchpoints above are not modified. `pdeq.json` is touched only to add the `harnesses` field when absent; existing fields are untouched.

The migration is idempotent: every branch short-circuits when its destination state is already correct. Re-running on an already-migrated project produces no further file changes.

## Mechanical

```shell
# Bring a 0.3.x consumer install to the 0.4.0 layout in one pass.
#
# This block is idempotent. Each branch checks for the destination state
# and skips when already correct, so re-running after a successful apply
# is a no-op:
#   - Lane CLAUDE.md @import targets: rewrite only when target points at
#     CLAUDE.md, leave alone when already at AGENTS.md.
#   - Non-Claude harness AGENTS.md symlinks: create only when absent.
#   - .claude/commands/pdeq-*.md symlinks: re-point only when target
#     still references the old .claude/commands/ path inside .pdeq/.
#   - .claude/agents/bootstrap-* entries: rm only when entry is a symlink
#     (pdeq-managed); regular files trigger a warning and are left alone.
#   - pdeq.json harnesses field: add only when absent.

set -eu

# ─── 1. Resolve consumer's harnesses list ────────────────────────────────
# Default to ["claude"] when pdeq.json has no harnesses field — this is the
# v0.3.x back-compat case where the consumer never declared a list.
HARNESSES_LIST=""
if [ -f pdeq.json ]; then
  raw=$(tr -d '\n' < pdeq.json \
        | grep -oE '"harnesses"[[:space:]]*:[[:space:]]*\[[^]]*\]' \
        | head -n1 || true)
  if [ -n "$raw" ]; then
    inner=${raw#*\[}
    inner=${inner%\]}
    HARNESSES_LIST=$(echo "$inner" | grep -oE '"[a-z][a-z0-9-]*"' | tr -d '"' | tr '\n' ' ')
  fi
fi
[ -z "$HARNESSES_LIST" ] && HARNESSES_LIST="claude"

HAS_CLAUDE=0
HAS_NONCLAUDE=0
for h in $HARNESSES_LIST; do
  case "$h" in
    claude) HAS_CLAUDE=1 ;;
    codex|pi) HAS_NONCLAUDE=1 ;;
  esac
done

# ─── 2. Rewrite per-lane CLAUDE.md @import targets + add AGENTS.md ───────
# Lanes: root + each functional-area folder under specsRoot. The runner
# invokes this block with cwd = consumer git root, so paths are relative.
for lane in . product design engineering qa roadmap; do
  [ -d "$lane" ] || continue

  if [ "$lane" = "." ]; then
    claude_file="./CLAUDE.md"
    agents_target=".pdeq/AGENTS.md"
  else
    claude_file="./$lane/CLAUDE.md"
    agents_target="../.pdeq/$lane/AGENTS.md"
  fi

  # 2a. If CLAUDE.md is a pdeq-managed @import wrapper, rewrite target.
  if [ -f "$claude_file" ] && [ ! -L "$claude_file" ]; then
    first_line=$(head -n 1 "$claude_file" 2>/dev/null || true)
    case "$first_line" in
      @*.pdeq/*CLAUDE.md)
        # Replace the trailing CLAUDE.md with AGENTS.md on the import line.
        # Pattern: any path ending in `.pdeq[/...]?/CLAUDE.md`. The `[^[:space:]]*`
        # is greedy so it matches both root (`.pdeq/CLAUDE.md`) and per-lane
        # (`.pdeq/product/CLAUDE.md`) shapes.
        sed 's|\(\.pdeq[^[:space:]]*\)/CLAUDE\.md|\1/AGENTS.md|' \
          "$claude_file" > "$claude_file.tmp"
        mv "$claude_file.tmp" "$claude_file"
        ;;
    esac
    # else consumer-authored CLAUDE.md or already pointing at AGENTS.md — leave alone.
  fi

  # 2b. For non-Claude harnesses, create AGENTS.md symlink if absent.
  if [ "$HAS_NONCLAUDE" = "1" ]; then
    if [ "$lane" = "." ]; then
      agents_dest="./AGENTS.md"
    else
      agents_dest="./$lane/AGENTS.md"
    fi
    if [ ! -e "$agents_dest" ] && [ ! -L "$agents_dest" ]; then
      ln -s "$agents_target" "$agents_dest"
    fi
  fi
done

# ─── 3. Re-point .claude/commands/pdeq-*.md symlinks ─────────────────────
# The slash-command source moved from .pdeq/.claude/commands/ to
# .pdeq/pdeq-rules/commands/ inside the submodule. Existing symlinks
# in the consumer's project still reference the old path; rewrite each.
COMMANDS_DIR=".claude/commands"
if [ -d "$COMMANDS_DIR" ]; then
  for f in "$COMMANDS_DIR"/pdeq-*.md; do
    [ -L "$f" ] || continue
    name=$(basename "$f")
    target=$(readlink "$f")
    case "$target" in
      *.pdeq/.claude/commands/pdeq-*.md)
        # Old path — re-point. The new symlink target is two levels up
        # to git root, then into .pdeq/pdeq-rules/commands/.
        rm "$f"
        ln -s "../../.pdeq/pdeq-rules/commands/$name" "$f"
        ;;
      *.pdeq/pdeq-rules/commands/pdeq-*.md)
        # Already migrated — skip.
        : ;;
      *)
        # Consumer-customized target — leave alone.
        : ;;
    esac
  done
fi

# ─── 4. Remove deleted .claude/agents/bootstrap-* entries ────────────────
# Pre-0.4.0 init.sh symlinked these into the consumer's project. The
# subagent prompts are now inlined into pdeq-bootstrap.md, so the entries
# point at files that no longer exist in the submodule.
AGENTS_DIR=".claude/agents"
for sub in bootstrap-analyzer bootstrap-generator; do
  path="$AGENTS_DIR/$sub"
  if [ -L "$path" ]; then
    rm "$path"
  elif [ -d "$path" ] || [ -f "$path" ]; then
    printf '[pdeq 0.4.0] %s exists as a non-symlink (consumer customization) — leaving alone. Remove manually if no longer needed.\n' "$path" >&2
  fi
done
# Clean up empty .claude/agents/ dir if we emptied it.
if [ -d "$AGENTS_DIR" ]; then
  rmdir "$AGENTS_DIR" 2>/dev/null || true
fi

# ─── 5. Add harnesses field to pdeq.json if absent ───────────────────────
# Default to ["claude"] so existing Claude Code consumers see no behavioral
# change beyond the file-layout normalization. Consumers can edit the list
# later to add codex/pi.
if [ -f pdeq.json ] && ! grep -q '"harnesses"' pdeq.json; then
  # Insert harnesses field immediately before the final closing brace.
  # Strategy: find the last meaningful "}" line; emit a trailing comma on
  # the line before it (if it doesn't already have one), insert the
  # harnesses entry, then the closing brace. Works for the pdeq.json
  # shape init.sh produces.
  awk '
    {
      lines[NR] = $0
    }
    END {
      # Find the last line that is just "}" (possibly with leading whitespace).
      last_brace = 0
      for (i = NR; i >= 1; i--) {
        if (lines[i] ~ /^[[:space:]]*\}[[:space:]]*$/) {
          last_brace = i
          break
        }
      }
      if (last_brace == 0) {
        # No closing brace found — emit unchanged.
        for (i = 1; i <= NR; i++) print lines[i]
        exit
      }
      # Emit everything up to (but not including) the line before the brace.
      for (i = 1; i < last_brace - 1; i++) print lines[i]
      # The line just before the brace: append a comma if not already present.
      prev = lines[last_brace - 1]
      sub(/[[:space:]]*$/, "", prev)
      if (prev !~ /,$/) prev = prev ","
      print prev
      # Insert the harnesses field.
      print "  \"harnesses\": [\"claude\"]"
      # Emit the closing brace.
      print lines[last_brace]
      # Emit anything after the closing brace (trailing newline content, etc.).
      for (i = last_brace + 1; i <= NR; i++) print lines[i]
    }
  ' pdeq.json > pdeq.json.tmp
  mv pdeq.json.tmp pdeq.json
fi
```

## Notes

The migration's `scope` is `default`, which expands to the consumer's `specsRoot` plus `pdeq.json`. The block also writes inside `.claude/commands/` and `.claude/agents/`, which live at the git root rather than under `specsRoot`. For root-level installs (the common case where `specsRoot: "."`), the `default` scope already covers everything. For nested or specs-only installs whose `specsRoot` is a subdirectory, the `.claude/*` writes still happen at the git root because that is where pre-0.4.0 `init.sh` placed them, so the writes are unavoidable. The migration writes nothing under directories that pdeq has never previously managed.

Idempotency rests on per-branch destination checks:
- Lane `CLAUDE.md`: `case` matches `@.../CLAUDE.md`; after rewrite the first line ends in `AGENTS.md` and no longer matches, so the rewrite is skipped.
- AGENTS.md symlinks for non-Claude harnesses: `[ ! -e ] && [ ! -L ]` skips when present.
- `.claude/commands/pdeq-*.md`: `case` distinguishes old path (rewrites) vs new path (skips) vs consumer-customized (skips with no message).
- `.claude/agents/bootstrap-*`: `[ -L ]` removes only symlinks; regular files trigger a one-line warning and are left alone (matches the 0.3.0 migration's policy for customized commands).
- `pdeq.json` harnesses field: `grep -q '"harnesses"'` skips when the field is present.

The pdeq self-host case: pdeq's own repo has no `.pdeq/` submodule (the framework files live at the repo root). Branch 1 reads pdeq.json's existing harnesses; branch 2 finds `CLAUDE.md` at root as a regular file but its first line is `@AGENTS.md` (the wrapper created in PDEQ-fwfwxqch), not `@.pdeq/...CLAUDE.md`, so the rewrite skips. Branch 2b creates `AGENTS.md` symlinks — but they would point at non-existent targets in self-host. Self-host is expected to have `selfHost: true` in pdeq.json and to set `harnesses: ["claude", "codex"]` explicitly (PDEQ-dxmwmvtw); the migration's branch-skip behavior on already-correct state handles re-runs without harm. Branches 3 and 4 are no-ops because the self-host's `.claude/commands/` symlinks already point at `pdeq-rules/commands/` (set up in PDEQ-fwfwxqch) and `.claude/agents/` was already removed in the same commit. Branch 5 is a no-op because pdeq self-host's `pdeq.json` already has `"harnesses"` (added in PDEQ-akjxdidz).
